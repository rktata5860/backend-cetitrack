# CertiTrack Backend (Spring Boot + MySQL)

A separate Spring Boot backend service for CertiTrack admin analytics and user/certificate tracking.

## Tech Stack
- Java 17
- Spring Boot
- Spring Web
- Spring Data JPA
- MySQL Driver
- Lombok
- Maven

## MySQL Tables (Generated by JPA)

### 1) users
- `id` (PK)
- `name`
- `email` (unique)
- `role` (`USER`/`ADMIN`)
- `created_at`
- `last_login`
- `login_count`
- `status` (`ACTIVE`/`INACTIVE`)

### 2) login_history
- `id` (PK)
- `user_id`
- `user_name`
- `email`
- `login_time`

### 3) certificates
- `id` (PK)
- `user_id`
- `title`
- `issuer`
- `category`
- `issue_date`
- `expiry_date`
- `credential_id`
- `credential_url`
- `notes`
- `status` (`VALID`/`EXPIRING`/`EXPIRED`)

## API Endpoints

### Users
- `POST /api/users/register`
- `POST /api/users/login`
- `GET /api/users`
- `GET /api/users/count`
- `GET /api/users/logged-in`

#### Register sample payload
```json
{
  "name": "Alice",
  "email": "alice@example.com",
  "role": "USER"
}
```

#### Login tracking sample payload
```json
{
  "name": "Alice",
  "email": "alice@example.com",
  "role": "USER"
}
```

### Login History
- `GET /api/logins`

### Certificates
- `POST /api/certificates`
- `GET /api/certificates`
- `GET /api/certificates/{id}`
- `PUT /api/certificates/{id}`
- `DELETE /api/certificates/{id}`
- `GET /api/certificates/user/{userId}`

#### Certificate create/update sample payload
```json
{
  "userId": 1,
  "title": "AWS Cloud Practitioner",
  "issuer": "Amazon",
  "category": "Cloud",
  "issueDate": "2026-01-10",
  "expiryDate": "2029-01-10",
  "credentialId": "AWS-123456",
  "credentialUrl": "https://example.com/cert/aws-123456",
  "notes": "First cloud certificate",
  "status": "VALID"
}
```

### Admin Dashboard
- `GET /api/admin/dashboard`

Returns:
- `totalRegisteredUsers`
- `totalLoggedInUsers`
- `totalLogins`
- `totalCertificates`
- `validCertificates`
- `expiringCertificates`
- `expiredCertificates`
- `recentLogins`

## application.properties

```properties
spring.application.name=certitrack-backend
server.port=9091

spring.datasource.url=jdbc:mysql://localhost:3306/aidss2
spring.datasource.username=root
spring.datasource.password=YOUR_PASSWORD
spring.datasource.driver-class-name=com.mysql.cj.jdbc.Driver

spring.jpa.hibernate.ddl-auto=update
spring.jpa.show-sql=true
spring.jpa.properties.hibernate.format_sql=true
spring.jpa.database-platform=org.hibernate.dialect.MySQL8Dialect
```

## Run in VS Code / STS

1. Open this folder as a Maven project.
2. Update MySQL credentials in `src/main/resources/application.properties`.
3. Create DB if needed:
   ```sql
   CREATE DATABASE aidss2;
   ```
4. Run app:
   - VS Code: Run `CertitrackBackendApplication`
   - STS: Right click main class -> Run As -> Spring Boot App
5. Or run with Maven:
   ```bash
   mvn spring-boot:run
   ```

## What you should see in MySQL

After registration (`/api/users/register`):
- One row in `users` with `created_at` set, `login_count = 0`, status `ACTIVE`.

After login tracking (`/api/users/login`):
- Existing user: `last_login` updated and `login_count` incremented.
- New user: inserted with `created_at`, `last_login`, and `login_count = 1`.
- Every login adds one row into `login_history`.

This backend is separate from frontend auth and is safe to use for admin analytics via Postman.
